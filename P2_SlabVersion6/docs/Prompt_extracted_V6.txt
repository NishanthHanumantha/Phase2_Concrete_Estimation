================================================================================
MISSION (unchanged from V5 docx)
================================================================================
Redesign SDIE into a Structural Reasoning Engine that interprets structural
drawings the way an experienced structural estimator does.

V4 foundations: Component Atlas, Layer Profiles, DeepSeek ambiguity resolution,
Structural Graphs, Knowledge Base.

V5 adds: estimator-style multi-evidence reasoning, topology-enriched DeepSeek
prompts, review queue, semantic building model.

V5/V6 implementation adds: slab intelligence with gated exclusions, estimator
bay merge, multi-panel floor zones, Inizio podium grid, project-aware soft-layer
arbitration.

V6 adds: drawing-driven layer discovery (no hard-coded S_FRAMES default on
Inizio sheets).

V6 (Revised Knowledge): INIZIO teaching corpus is reorganized under
Revised Project Knowledge/Tagged files/. The model must learn how the
estimator draws closed polylines on the raw structural plan and assigns
component tags (Slab, Beam, Column, Shear Wall) matching the four
supervised tagged references. All downstream pipeline steps are unchanged.

The objective is structural understanding first; slab quantities are
deterministic and run only after classification + exclusions.

================================================================================

================================================================================
Increase component classification accuracy from 50–60% toward 85–95%.

Primary target classes: Slab, Beam, Column, Shear Wall
Supporting: Structural Wall, Lift Core, Stair Core, Shaft, Opening, Unknown

Business calculators (deterministic, post-classification):
Slab, Beam, Column, Shear Wall — all other types support exclusions/diagnostics.

  Input  → Raw structural DXF (frame lines, annotations — no estimator tags)
  Teach  → Four tagged DXFs (estimator-drawn polylines + component labels)
  Output → On raw file: infer the same slab bays, beam runs, column footprints,
           and shear-wall regions the estimator tagged in the teach set

LAYER DISCOVERY ENGINE (V6 — NEW)
================================================================================
Problem (V5): CLI defaulted to --layers S_FRAMES STR-CUTOUT, which returns
zero slabs on Inizio drawings that use S-BEAM for both beams and perimeter walls.

Approach: single modelspace scan + project profile boost before classification.

Module: src/sdie/ingestion/layer_discovery.py

Algorithm:
1. scan_layer_stats(msp) — per-layer counts of LINE, LWPOLYLINE, TEXT, HATCH
2. Score structural candidates:
   - lines × 3 + polylines × 2 + hatches
   - × project layer_profiles boost for (project_id, layer)
   - × 1.8 if layer name matches BEAM|FRAME|STR-BEAM|S-BEAM pattern
   - down-rank cutout/annotation-dominant layers
3. Keep candidates ≥ max(25% of top score, 30)
4. Annotation layers: any layer with TEXT/MTEXT or ANNO|IDEN|FLOR|THK in name
5. Cutout layers: STR-CUTOUT, SUNK SLAB, S-SUNKEN HATCH, name match CUTOUT|VOID
6. frame_layers: top 1–2 line-heavy structural layers (for beam-grid + floor zone)

Resolved sets written to PipelineConfig and logged in results JSON as layer_discovery.

Validated auto-discovery:
  INIZIO  → frame_layers: [S-BEAM], structural: [S-BEAM, SUNK SLAB]
  TRUST_OFFICE → frame_layers: [S_FRAMES], structural: [S_FRAMES, STR-CUTOUT, S-SUNKEN HATCH]

CLI override: --layers S-BEAM --no-auto-layers (disables discovery for framing)

Diagnostic: scripts/analyze_dxf.py --project-id <ID> prints v6_layer_plan

CLASSIFICATION STACK (V4 + V5)
================================================================================
Order of precedence:
1. Void/THK/beam-tag annotation rules
2. Soft-layer geometry arbitration (see below)
3. Per-project layer profiles (data/layer_profiles.json)
4. Hard global layer hints (S-COLS, STR-CUTOUT, A-FLOR-IDEN, …)
5. Geometry classify (beam_enclosure, column_compactness, wall_continuity)
6. Atlas k-NN vote (layer-indexed)
7. DeepSeek v5 for ambiguous entities only (threshold 75%)

Soft layers (geometry required — never hard-map layer name alone):
  S_FRAMES, STR-BEAM, S-BEAM, S-BEAM-IDEN

Soft-layer arbitration (INIZIO S-BEAM — IMPLEMENTED):
  If geometry:beam_line with conf ≥ 0.88 → Beam (interior grid lines)
  Elif layer_profile Shear Wall with conf ≥ 0.85 → Shear Wall (perimeter)
  Elif geometry wall_line / wall_polyline ≥ 0.82 → Shear Wall
  Else fall through to project profile rule

Evidence tags for downstream slab exclusions:
  layer_profile:<layer>:<entity_type>
  geometry:beam_line | geometry:wall_line | geometry:wall_polyline
  (NOT geometry:wall — must match classifier output exactly)

V5 multi-evidence weights (src/sdie/confidence/scorer.py):
  Layer 25% | Geometry 25% | Topology 20% | Annotation 15% | Atlas 10% | KB 5%
  Thresholds: auto_accept ≥90, review ≥75, force_queue <60

Note: blended V5 scores understate high layer-profile confidence (~89% rule
→ ~56% final). Exclusion gating uses effective confidence floor for profile-
backed frame-layer shear walls (see Semantic Exclusions).

FLOOR ZONE ENGINE (multi-panel DXF — IMPLEMENTED)
================================================================================
Module: src/sdie/detection/floor_zone.py

Purpose: stacked plans on one sheet (tower + basement satellite + duplicate copy).

Method: thk_cluster on A-FLOR-IDEN *THK labels
  - Primary cluster = largest THK band (B2 tower on Inizio)
  - Satellite clusters merged only if within gap of primary (not far-below stacks)
  - Exclude Basement-03 tags far below primary (y < primary.y_min - 0.35×gap)
  - Subpanel split when duplicate tower copy detected (internal Y gap)
  - grid_bounds_y capped at selected subpanel ymax
  - frame ymin extends floor bounds to lowest S-BEAM/frame line (podium margin)

Outputs: bounds_y (grid), label_bounds_y (THK filter), notes in results JSON

SLAB INTELLIGENCE ENGINE v3.3 (IMPLEMENTED)
================================================================================
Module: src/sdie/detection/slab_intelligence.py
Never identify slabs first. Classify → exclude → derive framed bays.

Strategy selection (priority):
1. semantic_label_merged_bay — *THK labels merged when ≥ min_thk_labels_for_merge
2. semantic_beam_grid_bay[_merged] — orthogonal beam centerline grid
3. +podium_grid — supplementary bays below THK band (Inizio podium)

Config highlights (src/sdie/config.py):
  min_slab_area_m2: 0.4
  merge_beam_grid_to_estimator_bays: true
  apply_slab_exclusions: true
  use_layer_gated_semantic_exclusions: true
  enable_podium_beam_grid_slabs: true
  frame_layer_shear_wall_exclusion_min_confidence: 55

Beam grid (detection/beam_grid.py):
  - Collect orthogonal LINE axes on frame_layers
  - Line intersects Y band (not centroid-only) for vertical beams in podium band
  - augment_sparse_grid_axes: add band edges + primary X cluster for sparse grids
  - ExclusionCatalog difference per cell before area threshold

Label merge (detection/slab_by_label.py):
  - Merge *THK TEXT/MTEXT into estimator bays using grid_cells as seeds
  - bounds_y = label_bounds_y from floor zone

================================================================================
ESTIMATOR BAY MERGE (IMPLEMENTED — Trust Office S43 fix)
================================================================================
Module: src/sdie/detection/bay_merge.py

Merges adjacent raw beam-grid cells into physical estimator bays.

Key behaviours:
  - infer_from_cells: auto span/area caps from cell population
  - Column-stack merge: vertical neighbours sharing X column (i fixed, j±1)
  - _span_touch_or_overlap: allow merge when grid lines touch/overlap (not only
    identical edge coordinates) — fixes 1 m sliver bays blocking 27 m² merge
  - _column_stack_pair / _is_row_sliver: stack thin row slivers without bridging
    full adjacent bays
  - Elevated area caps only for column-stack merges (not global)
  - auto_inferred=True preserves infer_from_cells caps when merging

Regression: tests/test_bay_merge.py (includes S43-style vertical stack)

================================================================================
PODIUM BEAM GRID (INIZIO — IMPLEMENTED)
================================================================================
Band: ymin = floor_bounds_y[0] to ymax = thk_label_bounds_y[0] - gap_mm

_infer_podium_bounds_y:
  Do NOT collapse ymax to tight horizontal-beam cap when framing is sparse
  (Inizio corner detail at y≈1–3 m below full 15 m band).
  Use ymax_candidate when horizontal span < 12% of band height.

Sparse grid helpers (beam_grid.py):
  - _select_primary_vertical_axis_cluster: drop duplicate plan X copies
  - _augment_horizontal_axes / _augment_vertical_axes for 1–2 axis grids
  - min_vertical_span uses Y-band overlap length (tower walls crossing band)

Typical Inizio result: 2 podium cells ~7.7 m² at low Y + tower label-merge bays

================================================================================
SEMANTIC EXCLUSION GATING (IMPLEMENTED)
================================================================================
Module: slab_intelligence._classified_exclusion_geoms + _layer_allows_semantic_exclusion

Geometric exclusions: columns, hatches, void labels, cutouts (layer lists in config)

Semantic exclusions (classified components):
  Gated by layer + confidence + evidence

Frame-layer shear walls on S-BEAM (Inizio):
  Allow exclusion when:
    - component_type Shear Wall or Structural Wall
    - layer in frame_layers
    - evidence starts with layer_profile: OR geometry:wall_line OR geometry:wall_polyline
    - effective confidence ≥ frame_layer_shear_wall_exclusion_min_confidence (55)
      OR layer_profile boost to 88 for gating (V5 blend understates rule conf)

Do NOT deduct interior beam centerlines on frame_layers (Beam classification).

Buffer widths: wall 200 mm, column 50 mm, core 4500 mm, opening 1500 mm

Results JSON fields:
  semantic_exclusion_area_m2, semantic_exclusion_parts, total_exclusion_area_m2
