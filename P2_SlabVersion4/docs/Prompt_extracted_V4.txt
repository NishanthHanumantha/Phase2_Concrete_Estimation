SDIE VERSION 4 – DEEPSEEK SEMANTIC STRUCTURAL INTELLIGENCE ENGINE
MISSION
Redesign the current SDIE architecture from a geometry-first slab detection system into a semantic structural intelligence platform capable of learning from completed estimator projects.
The primary failure in Version 3 is incorrect slab identification.
The system incorrectly classifies:
Beams
Columns
Structural Walls
Shear Walls
Lift Cores
Stair Cores
Shafts
Openings
as slab regions.
The new system must perform structural understanding before quantity estimation.

BUSINESS OBJECTIVE (COMPANY WORKFLOW)
Slab, Beam, Column, and Shear Wall quantities are calculated separately.
The generic model must reliably identify these four component families before any quantity engine runs.
Internal classes (Structural Wall, Lift Core, Opening, etc.) support exclusion and diagnostics but map to the four business calculators:
  Slab → Slab calculator
  Beam → Beam calculator
  Column → Column calculator
  Shear Wall / Structural Wall (structural) → Shear Wall calculator
  Openings / cores / shafts → Exclusions (not slab area)

OBJECTIVE
Enable the system to learn from 3 completed projects initially (Inizio, Trust Office, Manohar) and expand to 5+ projects later.
The system must ingest:
Raw structural DXF drawings
Tagged beam DXF drawings
Tagged slab DXF drawings
Tagged column DXF drawings
Tagged shearwall DXF drawings
Estimator quantity workbook
Component mapping information (projects_manifest.json)
and build a permanent structural knowledge base.

CORE PRINCIPLE
The current system is geometry-driven.
Version 4 must become knowledge-driven.
The system should learn how estimators identify structural components and how they distinguish slabs from non-slab elements.
The estimator workbook is the slab quantity ground truth source.
Tagged component DXF drawings are the entity-level semantic truth source for Beam / Column / Shear Wall.
The combination of both forms the training and evaluation corpus.

ARCHITECTURE
Raw DXF
→ Geometry Extraction (entity_extractor)
→ Feature Extraction (geometry + annotation features)
→ Rule Baseline + Per-Project Layer Profiles
→ Component Knowledge Base (RAG) + Component Atlas (supervised + rule samples)
→ DeepSeek Semantic Classification (ambiguous entities only)
→ Structural Graph Engine
→ Semantic Building Model
→ Slab Intelligence Engine (exclusions → framed bays → quantities)
→ Deterministic Quantity Engine
→ Validation Layer (entity classification eval + slab GT + diagnostic overlay)

IMPLEMENTED PIPELINE ORDER (P2_SlabVersion4)
1. Extract entities from structural + annotation layers
2. Classify components (rules → layer profiles → geometry → atlas → DeepSeek)
3. Build structural graph from classified components
4. Detect slab bays after non-slab exclusions (beam-grid + estimator bay merge)
5. Assign thickness labels and compute deterministic quantities
6. Export results JSON, building model, summary, diagnostic overlay HTML

DEEPSEEK MODEL STRATEGY
Primary Model:
DeepSeek-V3 (deepseek-chat via API)
Purpose:
Component Classification (ambiguous entities only)
Annotation Reasoning
Ambiguity Resolution
Layer Interpretation
Estimator Pattern Recognition
Structural Context Understanding
DeepSeek must never calculate:
Area
Volume
Concrete Quantity
Shuttering Quantity
All calculations must remain deterministic and auditable.

CLASSIFICATION RULE STACK (IMPLEMENTED – PHASE A)
Classification is no longer a single global layer map. Order of precedence:
1. Annotation rules (THK → Slab, void keywords → cores/openings, beam size tags → Beam)
2. Hard global layer rules (high confidence, project-independent):
   S-COLS, S-COL HATCH, S-COL → Column
   S-BEAM, S-BEAM-IDEN → Beam
   S-SHEARWALL, S-SHEAR, S-WALL → Shear Wall
   A-FLOR-IDEN → Slab (annotation context)
3. Per-project layer profiles (learned from supervised teaching DXFs)
   data/layer_profiles.json — keyed by (project_id, layer[, entity_type])
4. Geometry rules for soft/ambiguous layers (S_FRAMES, STR-BEAM):
   Geometry decides Column vs Beam vs Wall when layer name is shared
   Example: Trust Office uses S_FRAMES for columns on column-tagged drawings
5. Atlas k-NN vote (layer-indexed, supervised samples weighted at confidence 1.0)
6. DeepSeek + RAG (only if entity still ambiguous below confidence threshold)

Soft layers (geometry required before type assignment):
  S_FRAMES, STR-BEAM
Hard layers (type from layer name when entity type matches):
  S-COLS, S-COL HATCH, S-BEAM, S-SHEARWALL, etc.

High-confidence rule/profile hits (≥82%, evidence: hard_layer: or layer_profile:) skip DeepSeek.

COMPONENT KNOWLEDGE BASE (RAG)
Permanent retrieval-based structural knowledge base at data/knowledge_base/structural_kb.json
Built by: scripts/build_knowledge_base.py (after ingest + atlas)
Store:

  GLOBAL hard layer hints
  Per-project layer profiles (from supervised DXFs)
  Atlas-derived layer → type mappings
Annotation Knowledge
  THK patterns, stair/lift/shaft keywords, default slab notes
Component Pattern Knowledge
  Geometry features (aspect ratio, entity type, area buckets) per project
Estimator Mapping Knowledge
  Workbook totals → drawing signals (from data/ground_truth/*.json)

SLAB INTELLIGENCE ENGINE
Never identify slabs first.
Always identify and exclude non-slab components, then detect framed slab bays.
IMPLEMENTED DETECTION IMPROVEMENTS:
  min_slab_area_m2 default: 0.4 m² (was 10 m² — recovered small GT bays)
  merge_beam_grid_to_estimator_bays: true
  Estimator-level bay merge (src/sdie/detection/bay_merge.py):
    Merges adjacent partial beam-grid cells into estimator-sized bays
    Fixes under-segmentation when beam axes split one physical bay
  Beam footprints frame bays; columns/cores/walls are exclusions (beams not deducted)
Remaining slab GT gaps: edge ledges, large irregular bays — ongoing tuning.

DEEPSEEK CLASSIFICATION WORKFLOW
Input per ambiguous entity:
  Geometry features, layer, annotation text, rule baseline type/confidence
  RAG context (layer knowledge, annotation rules, atlas samples, patterns)
Output:
  {"component_id": "ENT-00001", "component": "Beam", "confidence": 0.96,
   "evidence": ["Layer Match", "Atlas Match"]}
Batch size: 30 entities; only ambiguous entities sent (not entire drawing when rules are confident).

CONFIDENCE FRAMEWORK
Final Confidence Score:
  35% Geometry (rule/profile/atlas baseline)
  25% Topology
  20% Structural Graph
  20% DeepSeek Semantic Confidence
Output: 0–100%
Threshold: 65% default; entities below → DeepSeek candidate.

QUANTITY ENGINE
Deterministic only. src/sdie/quantity/slab.py
Concrete: Area × Thickness
Shuttering: Area (configurable soffit equality)
All calculations traceable in calculation_trace per slab.

GROUND TRUTH CONVENTIONS
GT (Ground Truth) = expected correct answer from estimator
  Slab GT: Excel workbook quantities (count, area, concrete)
  Component GT: isolated tagged DXF drawings (all entities = that component type)
Do not compare slab output to workbook unless GT JSON/Excel is provided for that drawing.
